核心功能

网页访问

通过代理直接访问 GitHub 网页，浏览仓库、查看代码和 Issues，无需额外配置。

代码克隆

加速 git clone 操作，快速获取代码仓库，提升项目初始化效率。

推送代码

支持 git push 操作，轻松提交代码更改到 GitHub 仓库，无需担心连接问题。

使用方法

1 网页访问

直接在浏览器中访问：

https://你的域名/api/proxy/用户名/仓库名

示例：https://你的域名/api/proxy/tensorflow/tensorflow

2 Git 克隆仓库

使用代理克隆 GitHub 仓库：

git clone https://你的域名/api/proxy/用户名/仓库名.git

示例：git clone https://你的域名/api/proxy/vuejs/vue.git

3 配置 Git 推送代理

设置全局代理（所有仓库生效）：

git config --global http.https://github.com.proxy https://你的域名/api/proxy

仅对当前仓库设置代理：

git config http.https://github.com.proxy https://你的域名/api/proxy

常见问题

由于 Vercel 服务器函数有超时限制（通常为 10 秒），对于超过 100MB 的大文件传输可能会失败。建议大文件使用 Git LFS 或通过本地代理工具传输。

对于私有仓库，你需要使用个人访问令牌（PAT）进行认证。可以在克隆仓库时使用以下格式：

git clone https://<用户名>:<PAT>@你的域名/api/proxy/用户名/仓库名.git

个人访问令牌可以在 GitHub 的设置中创建，需要包含 repo 权限。

本代理基于 Vercel 免费计划提供，受其服务条款限制，包括：

  • 每小时最多 1000 次函数调用
  • 每次请求最长 10 秒超时
  • 带宽限制

建议仅用于个人开发用途，避免大规模商业使用。

如果代理连接失败，可以尝试以下步骤：

  • 检查 Vercel 部署状态是否正常
  • 确认输入的仓库路径是否正确
  • 尝试清除浏览器缓存或使用隐私模式
  • 检查网络环境是否允许访问 Vercel 服务

如果问题持续，可以查看 Vercel 控制台的函数日志获取详细错误信息。
